# VPN Connectivity Issues

## What Is A VPN?

A VPN (Virtual Private Network) creates a secure encrypted connection between a user's device and a private company network.

VPNs allow employees to securely access company resources when working remotely.

Examples of resources accessed via VPN:

- File servers
- Internal applications
- Shared drives
- Company intranet
- Remote desktop services

---

## Common VPN Issues

### User Cannot Connect To VPN

Symptoms:

- VPN login fails
- Connection times out
- User receives authentication errors
- VPN disconnects immediately

---

## Troubleshooting Process

### 1. Determine Scope

Ask:

- Is the issue affecting one user or multiple users?
- Have other users reported the same issue?

If multiple users are affected, the issue may be with the VPN service itself.

---

### 2. Verify Internet Connectivity

Check whether the user can:

- Browse websites
- Access public internet services

A VPN connection requires a working internet connection.

---

### 3. Check Credentials

Verify:

- Username
- Password

Ask:

- Has the password recently changed?

Old credentials are a common cause of VPN failures.

---

### 4. Check MFA

Many VPNs use Multi-Factor Authentication.

Confirm:

- MFA prompts are appearing
- Authenticator app is working
- User is approving requests correctly

---

### 5. Review Error Messages

Gather the exact error.

Examples:

- Authentication failed
- Unable to reach VPN server
- Connection timeout
- Certificate error

Error messages often point directly to the root cause.

---

### 6. Check Recent Changes

Ask:

- New laptop?
- New phone?
- Password recently changed?
- Home broadband issues?
- New VPN client version?

Recent changes often explain the problem.

---

### 7. Restart VPN Client

Basic step:

1. Close VPN application.
2. Reopen application.
3. Attempt connection again.

---

### 8. Restart Device

If appropriate:

1. Reboot device.
2. Retry connection.

This can resolve temporary software issues.

---

## Common Causes

### Authentication Issues

Examples:

- Incorrect password
- Expired password
- MFA failure
- Account lockout

---

### Network Issues

Examples:

- No internet access
- Unstable Wi-Fi
- ISP problems
- DNS issues

---

### VPN Service Issues

Examples:

- VPN server outage
- Maintenance window
- Backend infrastructure issue

Usually requires escalation.

---

## Escalation Criteria

Escalate if:

- Multiple users cannot connect
- VPN service appears unavailable
- Certificate issues are identified
- Firewall issues are suspected
- Administrative access is required

---

## Interview Scenario

### User Reports:

"I can't connect to the VPN."

Good response:

> First I'd determine whether the issue affects only that user or multiple users. I'd then verify internet connectivity, check credentials and MFA, gather any error messages, and identify whether the issue is authentication-related, network-related, or a VPN service issue before escalating if required.

---

## Interview Summary

A strong 1st line explanation:

> A VPN provides a secure encrypted connection to company resources when working remotely. Common troubleshooting steps include checking internet connectivity, credentials, MFA, error messages, and determining whether the issue affects a single user or multiple users before escalating if necessary.
