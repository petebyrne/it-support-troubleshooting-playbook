# New Starter Onboarding

## What Is New Starter Onboarding?

New starter onboarding is the process of preparing a new employee's accounts, devices, permissions, and access before their first day.

The goal is to ensure the user can be productive from day one.

---

## Typical Onboarding Tasks

### User Account Creation

Create accounts in:

- Active Directory
- Microsoft 365
- Email systems
- Business applications

Typical information required:

- Full name
- Job title
- Department
- Manager
- Start date

---

### Assign Permissions

Grant access to:

- Shared drives
- Teams channels
- SharePoint sites
- Department applications
- Security groups

Permissions should follow company policy and approved access requests.

---

### Hardware Preparation

Prepare:

- Laptop
- Desktop
- Monitor
- Docking station
- Headset
- Mobile phone (if required)

Tasks may include:

- Imaging
- Software installation
- Device configuration
- Security updates

---

### Microsoft 365 Setup

Configure:

- Outlook mailbox
- Teams account
- OneDrive
- MFA

Verify:

- Login works successfully
- Email can send and receive
- Teams access functions correctly

---

### Security Setup

Configure:

- Initial password
- MFA registration
- Security policies
- Device compliance requirements

Security should be completed before access is granted.

---

## Pre-Start Checklist

Before the user's first day:

- Account created
- Email active
- Required permissions assigned
- Hardware prepared
- Software installed
- MFA configured
- Documentation completed

---

## First Day Checks

Confirm the user can:

- Log into Windows
- Access Microsoft 365
- Send and receive email
- Access required applications
- Access required shared drives
- Use Teams successfully

---

## Common New Starter Issues

### User Cannot Log In

Check:

- Username
- Password
- Account status
- MFA setup

---

### Missing Access

Check:

- Group memberships
- Security permissions
- Application licensing

---

### Missing Software

Check:

- Deployment status
- Software assignment
- Installation completion

---

## Escalation Criteria

Escalate if:

- Required permissions are unclear
- Application licensing issues occur
- Administrative approval is required
- Hardware provisioning cannot be completed
- Security concerns are identified

---

## Interview Scenario

### Question

What would you do when onboarding a new starter?

### Good Answer

> I'd ensure the user account is created, required permissions and group memberships are assigned, hardware is configured, Microsoft 365 access is working, MFA is set up, and the user can successfully access the systems they need to perform their role.

---

## Leaver Process

When an employee leaves:

- Disable accounts
- Remove access permissions
- Recover company equipment
- Protect company data
- Document actions taken

The aim is to prevent unauthorised access after employment ends.

---

## Interview Summary

A strong 1st line explanation:

> New starter onboarding involves preparing user accounts, permissions, Microsoft 365 access, devices, and security settings before the employee's first day. The goal is to ensure they can securely access all required systems and begin working immediately.
